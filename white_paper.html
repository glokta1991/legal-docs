1. Identity & Network Architecture

  • Decentralized Identity (BIP39): Identity is founded on a 12-word Mnemonic seed generated locally. The developer never has access to this seed or the resulting private keys.
  • Blind Relay (HMAC Routing): Backend is a stateless "dumb" relay. Usernames are hashed locally via HMAC-SHA256 with quantum-hardened salts. The server knows "Packet A" goes to "Queue B" but has no link to real identity.
  • Zero-Discovery (Direct-Dial): Global user lists are destroyed. Search is exact-match only, preventing social graph enumeration.
  • Protocol Self-Healing: Automated handshakes silently re-negotiate secure tunnels if a session mismatch is detected, ensuring "Invisible Security."

2. At-Rest & Persistence Security

  • Encrypted Vault (SQLCipher): Local storage is protected by AES-256 byte-level encryption, accessible only via hardware-backed keys.
  • Header Kill-Switch (Scorched Earth): Pre-Init audit physically destroys the database if a plaintext header or identity mismatch is detected.
  • Active Data Decay: Messages are tagged with timestamps. A background engine overwrites expired data with random noise before physical purging.
  • Local Identity Map: Human-readable names exist only on the user's encrypted disk; the server only processes "User_xxxx" placeholders.

3. Forensic & Memory Hardening

  • Zero-String Policy: Database passphrases are handled exclusively as ByteArrays to avoid the immutable Java String Pool.
  • RAM Sanitization: Arrays.fill(key, 0.toByte()) scrubs sensitive keys from RAM the millisecond the handshake is complete.
  • TEE-Bound Security: Master keys are physically wrapped by the Secure Element (SE/TEE). Biometric signatures are required to unlock the vault for a narrow window.
  • Cloud Cord Cut (P2P Avatars): Profile pictures are shared strictly Peer-to-Peer as Base64 blobs inside the encrypted Signal envelope.

4. Lifecycle & Physical Guarding

  • Screen Privacy: Enforces FLAG_SECURE to block screenshots and masks app content in the Android task switcher.
  • Dual-Path Logout:
    • Nuclear (Secure Wipe): Total forensic destruction of all local keys and tables.
    • Standard: Session lock that preserves Identity Keys while scrubbing RAM.
  • Client-Side Metadata Guard: Masked WebSocket URLs and redacted logs prevent metadata capture by the OS or server.
