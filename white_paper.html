1. Identity & Network Architecture

  • Decentralized Identity (BIP39): SecureMessenger eliminates the need for phone numbers. Identity is founded on a 12-word seed generated locally. The developer never has access to this seed.
  • Blind Relay (HMAC Routing): The backend is a stateless "dumb" relay. Usernames are hashed locally via HMAC-SHA256 with 512-bit quantum-hardened salts.
  • Zero-Discovery (Direct-Dial): Search is exact-match only, preventing userbase scraping.

2. At-Rest & Persistence Security

  • Encrypted Vault (SQLCipher): All local storage is protected by AES-256 byte-level encryption.
  • Header Kill-Switch: Pre-Init audit physically destroys the database if a plaintext header or mismatch is detected.
  • Active Data Decay: Messages are forensically shredded (overwritten with random noise) before physical purging.

3. Forensic & Memory Hardening

  • RAM Sanitization: Arrays.fill(key, 0.toByte()) physically scrubs keys from RAM immediately after use.
  • TEE-Bound Security: Master keys are wrapped by the hardware Secure Element (SE/TEE).
